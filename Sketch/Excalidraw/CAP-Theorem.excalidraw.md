What is Consistency?
Is a application write a = 5 in DB 1 in India, then when it read in DB 2 in US it should
get a = 5, this means after successful writes data should be consistence.

What is Availability?
In the above example, all the DB nodes should be available, meaning they should respond to the
application, data may or may not be consistent.

What is Partition Tolerance?
In the above example when DB node 1 in India is syncing/replicating it's data to DB node 2 in US,
and their connection breaks, still if the application is able to query the data and get result,
that is called partition tolerance. Meaning system is up. ^odmbFNur

Why all 3 properties of CAP cannot be used together.

The Conflict:
In the presence of a network partition (P), the system must choose between:

Consistency (C): Ensuring all nodes have the same data, which requires delaying or rejecting 
operations until the partition is resolved.

Availability (A): Continuing to serve requests by allowing operations on the nodes that are reachable
,even if data becomes inconsistent.

Since network partitions are unavoidable in distributed systems, you cannot 
achieve both Consistency and Availability at the same time during a partition.


Partition Tolerance is always used, trade off is made between Consistency or Availability  ^fJs50AQA
