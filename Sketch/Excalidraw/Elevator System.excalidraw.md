Scan Algorithm

If the elevator is going up it will fulfil only those request which are going up.
If user is at 2nd floor and wants to go up and elevator is at 5 floor it cannot fulfil
the request, but if user is on 8 floor it can.

Disadvantage of this algorithm is, elevator has to go till the top floor then only change direction
to accept request of the user's which want to go down and vice-versa. ^Xq2acReg

The above disadvantage is covered by LOOK Algorithm.

Suppose if the user is on the second floor and the elevator is also on second floor,
and got a request from user to go to 6th floor, it will go till 6th floor,fulfil the request.
Now it will look ahead and check if is there any more request ahead if not it will turn back, if 
it gets a request to do down. ^Ye3vCfnI

Data Structure that can be used in LOOK Algo.
1. MinHeap -> for UP
2. MaxHeap -> for Down
3. Queue -> to keep a track of pending request

If the elevator is going up and is at 5 floor and user is at 2nd floor and wants to go up,
LOOK algo cannot put this request in minHeap as the elevator is going up and is on 6 floor, so this 
request will be put in queue. But if the request came from 8 floor to go up, this request will be added
in minHeap and not in the pending task.

Similary for maxHeap will be used while going down. ^fZgsGTo6
