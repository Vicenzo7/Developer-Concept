In TCP, a virtual connection is established before data transfer. 
Data is sent as packets, with sequencing to ensure proper order, 
even if packets arrive out of sequence. Lost packets are retransmitted,
and each received packet is acknowledged. 

TCP also implements flow control and congestion control for efficient 
and reliable data transmission.

Flow control ensures the sender does not overwhelm the receiver by 
adjusting the data transmission rate based on the receiver’s capacity.
Congestion control prevents network overload by dynamically adjusting
the sending rate based on network conditions like packet loss or delay. ^aRxuRQiD

UDP ^dBC9B12I

In UDP, multiple connections are not required as it is a connectionless protocol. 
Data is sent in parallel as datagrams, and if a packet is lost, it is not retransmitted. 
UDP does not ensure data ordering or acknowledgment,
making it lightweight and faster compared to TCP.

UDP is commonly used in applications where speed and low latency are critical, such as 
online gaming, video calling, live streaming, DNS queries, and real-time IoT applications.
However, it sacrifices reliability, ordering, and error correction for performance. ^jKRQ98OW
